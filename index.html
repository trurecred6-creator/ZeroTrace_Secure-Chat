Detailed Solutions

How End-to-End Encryption Works (Simple Explanation)

  • Step 1: Sender types message
  • Step 2: Message is encrypted using AES-256 key
  • Step 3: Key is encrypted using receiver public key
  • Step 4: Encrypted message is sent
  • Step 5: Receiver decrypts key using private key
  • Step 6: Receiver decrypts message

Only receiver can read the message.

Self-Destruct Message Feature

Problem:

Messages remain stored and can be stolen.

Solution:
  • Each message has a timer
  • After timer expires, message deletes automatically
  • Message cannot be recovered normally

Benefit: Protects sensitive information.

Anonymous Communication Feature

Problem:

User identity can be tracked.

Solution:
  • Use anonymous ID instead of real identity
  • Hide sender information
  • Protect user privacy
Types:
  • Partial anonymity → nickname
  • Full anonymity → completely hidden identity

Message Integrity Protection

Problem:

Message can be modified during transmission.

Solution:

We use SHA-256 hashing.

How it works:
  • Hash is generated at sender
  • Receiver generates hash again
  • If both match → message safe
  • If mismatch → tampering detected

Group Chat Security Problem

Problem:
  • If one group key is compromised, all messages are exposed.
  • This is a major weakness in traditional systems.
Our Advanced Solution — Secret Sharing:

We use Adi Shamir's Secret Sharing technique.

How it works:
  • Group key is split into multiple parts
  • Each user gets one part
  • No single user has full key
  • Key is reconstructed only when needed

Benefit: If one user is hacked, group remains safe.

Distributed Backup Vault (Advanced Feature)

Problem:

Centralized backup server can be hacked.

Solution:
  • Backup stored in distributed encrypted vault
  • Backup split into multiple secure parts
  • Cannot be accessed easily

Benefit: Maximum backup security.

Tampering Detection and Alert System

If attacker modifies message:

  • System detects hash mismatch
  • User gets alert
  • Message rejected

Prevents fake message attacks.
